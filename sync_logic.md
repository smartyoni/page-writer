# 🔄 실시간 동기화 로직 가이드 (Sync Logic Guide)

이 문서는 **글작성 도우미 (Page-Writer)**의 크롬 확장프로그램과 모바일 PWA 환경 간 실시간 데이터 동기화 구현 방식에 대해 설명합니다.

---

## 1. 핵심 아키텍처: 단일 경로 동기화 (Unified Path)

과거의 복잡한 "싱크 키"나 "기기 ID" 시스템을 폐기하고, 모든 기기가 파이버베이스(Firestore) 내의 **고정된 단일 경로**를 바라보도록 설계되었습니다.

- **데이터 주소**: `users/default_user/documents/{document_id}`
- **장점**: 
  - 별도의 페어링 과정이 필요 없습니다. 
  - 앱을 설치하고 실행하는 즉시 동일한 글 목록이 나타납니다.
  - 로그인이나 인증 없이도 "나만의 공유 공간"처럼 작동합니다.

---

## 2. 환경별 최적화 전략

동일한 코드 베이스(Vite + React)를 사용하지만, 실행 환경에 따라 보안 및 서비스 워커 설정을 다르게 처리합니다.

### 🌐 라이브 사이트 (PWA / Vercel)
- **PWA 활성화**: `vite-plugin-pwa`를 통해 모바일 기기에서 앱으로 설치 가능합니다.
- **서비스 워커**: `main.jsx`에서 `https:` 프로토콜임을 확인한 후 `sw.js`를 등록합니다.

### 🧩 크롬 확장프로그램 (Chrome Extension)
- **보안 정책(CSP)**: `manifest.json`에 `host_permissions`와 `connect-src`를 설정하여 파이버베이스 서버와의 통신 차단을 막습니다.
- **에러 차단**: 확장프로그램 환경에서는 일반적인 웹 서비스 워커가 지원되지 않으므로, **동적 임포트(Dynamic Import)**를 통해 PWA 관련 코드가 실행되지 않도록 격리했습니다.

---

## 3. 데이터 호환성 및 실시간성 (Nexus Layer)

기존 **Nexus** 앱 데이터와의 호환성을 유지하면서 실시간 통신을 보장합니다.

### 데이터 매핑 (Compatibility Mapping)
- **내용**: `pages[0]` (Nexus) → `content` (현재 앱) 자동 변환
- **날짜**: `updatedAt` (숫자형 밀리초) ↔ `modifiedAt` (파이버베이스 타임스탬프) 상호 호환

### 실시간 수신 (`onSnapshot`)
- `App.jsx` 내의 리스너가 데이터베이스의 변화를 초 단위로 감시합니다.
- **정렬 기준**: `updatedAt` 필드를 기준으로 최신순 정렬하여, 레거시 데이터와 신규 데이터가 모두 리스트에 나타나도록 보장합니다.

---

## 4. 데이터 저장 흐름

1. **사용자 입력**: Tiptap 에디터에서 글을 수정합니다.
2. **트리거**: 0.5초간 입력이 멈추면(Debounce) `onUpdate` 함수가 실행됩니다.
3. **저장**: Firestore의 `updatedAt`과 `modifiedAt` 필드를 동시에 기록합니다.
4. **전파**: 서버에 저장된 즉시 연결된 모든 기기(PC, Mobile)의 `onSnapshot` 리스너가 호출되어 화면을 갱신합니다.

---

> [!TIP]
> **확장프로그램 수정 시 주의사항**
> 코드를 수정한 후에는 반드시 `npm run build`를 통해 `dist` 폴더를 갱신하고, 크롬 확장프로그램 관리자 페이지에서 **새로고침(↻)**을 눌러야 실제 기기에 반영됩니다.
