# 코딩 컨벤션 및 디자인 가이드

## 코딩 컨벤션

- **언어**: TypeScript 필수, `any` 타입 사용 금지
- **주석**: 한국어로 작성
- **네이밍**: 컴포넌트는 PascalCase, 훅은 use- prefix, 서비스 함수는 camelCase
- **상태 관리**: 로컬 상태는 useState, 전역 공유는 Context API
- **Optimistic Update**: CRUD 시 UI를 먼저 업데이트한 뒤 Firestore에 반영
- **에러 처리**: try-catch로 감싸고, 실패 시 Optimistic Update 롤백
- **메모 데이터**: 페이지 간 이동 시 데이터 무손실 보장 필수

## 컴포넌트 사용 지침

### 확인 모달 (Confirmation)
- 삭제 등 사용자 확인이 필요한 대부분의 동작은 화면 중앙 모달 대신 **트리거 버튼 옆의 컨텍스트 팝오버(Contextual Popover)** 형식을 우선 사용한다.
- **지능형 위치 조정**: 팝오버는 하단의 "컨텍스트 메뉴 배치 시스템" 규칙에 따라 화면 공간에 맞춰 위/아래로 동적 위치 조정이 되어야 한다.
- `window.confirm()`, `window.alert()` 등 브라우저 기본 다이얼로그 사용 금지

### 입력 모달
- URL 입력, 텍스트 입력 등은 **앱 자체 모달 컴포넌트**를 사용한다 (예: `UrlInputModal`)
- `window.prompt()` 사용 금지

### 상세화면 (MemoEditorPanel) 전용 규칙
- **푸터 구성**: 보기/편집 모드 모두에서 단일 행(Single Row) 레이아웃을 유지한다. (1~5 숫자 탭과 태그/수정/닫기 버튼이 한 줄로 표시됨)
- **목차(ToC) 버튼**: `bg-emerald-50 text-emerald-600` 색상의 그린 테마를 사용하며, 일반 버튼보다 1.2배 넓은 너비(`px-4`)를 유지한다.
- **계층형 목차**: 본문에서 **"※"**로 시작하는 줄을 추출하여 목차에 표시하되, 목차 내부에서는 기호를 숨기고 **트리 구조(`└─`)** 스타일의 계층 라인만 노출한다.

## 디자인 및 타이포그래피

- **기본 폰트**: **Pretendard (프리텐다드)**를 기본으로 사용하며, 한글 가독성을 위해 `letter-spacing: -0.01em`을 권장한다.
- **색상 체계**:
    - Primary: `#2563eb` (Indigo Blue 계열)
    - 텍스트: 완전한 검정 대신 `#1e293b` (Slate 800) 사용 권장
    - 배경: 연한 그레이(`#f8fafc`) 배경과 흰색 카드의 조합 사용
- **UI 스타일**:
    - **둥근 모서리**: 모든 카드와 버튼은 `rounded-xl` (12px) 이상의 곡률을 기본으로 한다.
    - **그림자**: 강한 테두리 대신 부드러운 `shadow-sm` 또는 `shadow-md`를 사용하여 입체감을 부여한다.
    - **여백**: 컴포넌트 간 충분한 여백(공간)을 두어 정보의 밀집도를 낮춘다.

### 플레이스홀더 및 빈 상태(Empty State)
- **시각적 규칙**: 입력된 데이터가 없는 빈 상태는 `text-gray-400` 색상하고 `italic` (기울임꼴) 스타일을 적용하여 실제 데이터와 명확히 구분한다.
- **구체적 가이드**: 단순 "내용 없음" 대신, 해당 필드에 입력할 내용을 구체적으로 안내하는 플레이스홀더를 제공한다.
- **일관성**: 편집 모드(`textarea`, `input`)와 보기 모드 모두에서 동일한 플레이스홀더 문구를 유지한다.

### 표준 메모 박스 (Standard Memo Field)
사용자가 "텍스트 상자 추가해줘" 또는 "메모 박스 추가해줘"라고 요청할 경우, 별도의 지시가 없으면 아래 설정을 기본값으로 사용한다.
- **편집 트리거**: 더블 클릭(onDoubleClick)으로 인라인 편집 모드 진입.
- **자동 확장**: 
  - 기본 높이: 3줄 (`min-height: 4.5rem`)
  - 최대 높이: 5줄 (`max-height: 7.5rem`)
  - 5줄 초과 시 내부 스크롤 발생.
- **플레이스홀더**: "고객 요청사항 기타 확인할 것" (기본값).
- **스타일링**:
  - 내용이 없을 때: `text-gray-400 italic` 적용.
  - 내용이 있을 때: `text-gray-800` 적용.
  - 배경 및 테두리: 해당 섹션의 테마색(Primary)을 border 색상으로 사용.
- **구성**: 상단에 아이콘(`fas fa-sticky-note`) + 라벨 + 하단 메모 영역의 카드 형태(`bg-gray-50 rounded-lg`).

## 레이아웃 및 UX 원칙
- **전체 브라우저 사용**: 앱은 항상 브라우저의 전체 너비와 높이를 사용하며, 페이지 전체 스크롤이 아닌 내부 컴포넌트별 스크롤을 권장한다 (`h-full`, `w-full` 기반 레이아웃).
- **모바일 우선**: 모든 핵심 기능은 모바일 환경에서 한 손으로 조작 가능해야 한다.
- **다크 모드 배려**: 현재 스타일가이드 계열이 다크 모드에서도 가독성이 좋은지 항상 고려한다.

## 상호작용 및 내비게이션 (Interaction & Navigation)

### 컨텍스트 메뉴 배치 시스템 (Context Menu Positioning)
사용자 편의성과 화면 활용도를 극대화하기 위해 **"지능형 우측 동적 팝업(Smart Right-Side Positioning)"** 패턴을 따른다.
- **가로 위치**: 항상 트리거 요소(불렛, 아이콘 등)의 **즉각적인 우측**에 나타난다 (`rect.right + 4px`).
- **세로 위치 (Dynamic)**:
  - **기본 (Top-aligned)**: 트리거 요소 아래 공간이 충분할 경우, 메뉴의 상단을 트리거에 맞춘다.
  - **공간 부족 시 (Bottom-aligned)**: 화면 하단 등 공간이 부족할 경우, 메뉴의 하단을 트리거에 맞춰 위로 펼쳐지게 한다.

### 3. 북마크 관리 (Bookmark Management)
- **사이트 이동**: 사이트명(라벨)을 클릭하면 새 탭에서 해당 URL이 열립니다.
- **편집 모달 호출**: 항목 앞의 **보라색 불렛(점)을 클릭**하면 편집 모달이 열립니다. (사이트명 우클릭으로도 호출 가능)
- **UI 간결화**: 사이트명 아래에 미리보기로 나오던 URL 주소는 표시하지 않으며, 사이트명만 노출하여 인덱스 같은 깔끔한 느낌을 유지합니다.
