# 업무 수행 및 배포 지침

## 업무 수행 지침 (Operational Principles)

### 1. 엄격한 범위 준수 (Strict Scope)
- 요청받은 작업만 수행한다. 추가적인 UI 개선, 리팩토링, 기능 확장은 사용자가 명시적으로 요청하기 전에는 절대 먼저 수행하지 않는다.
- "확인해줘", "체크해줘"라는 요청에는 확인 결과(확인되었는지여부)만 보고하며, 이 과정에서 코드를 수정하거나 기능을 덧붙이지 않는다.

### 2. 제안 및 승인 절차
- 도움이 될 것 같은 추가 작업이 있다면, 코드를 수정하기 전에 반드시 먼저 질문을 통해 의사를 묻고 승인을 얻는다. (예: "수정 중 배지를 추가하면 더 좋을 것 같은데 진행할까요?")

### 3. 모든 작업 시 브라우저 시뮬레이션 금지
- 브라우저 시뮬레이션(브라우저 도구 사용)은 원칙적으로 **금지**한다.
- "특히 확인해줘", "체크해줘" 등의 요청 시에는 철저히 **코드 분석**을 통해 빠르게 응답한다.
- 시각적 확인이 반드시 필요한 경우나 사용자가 명시적으로 요청한 경우에만 **사용자에게 먼저 제안하고 승인을 받은 후**에만 제한적으로 수행한다.

## 배포 절차

배포는 GitHub 레포지토리에 설정된 GitHub Actions 워크플로우(`deploy.yml`)를 통해 자동으로 진행됩니다.

1. 로컬에서 작업 후 검증 완료 (배포 전 `npx tsc --noEmit`으로 타입 체크 필수)
2. 변경 사항을 모두 커밋
3. `git push origin main` 명령어로 코드를 `main` 브랜치에 푸시
4. 코드가 푸시되면 자동으로 `Deploy to GitHub Pages` 액션이 실행되어 반영됨
